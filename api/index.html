API :: Overview

The API is up and running, but the matchmaking daemon is offline. The means that games cannot be created. Games start at 6pm, when the matchmaking daemon is booted.

The API for this game has been designed to be as simple as possible. The primary control flow of a simple bot should be composed of the following endpoints:

  • bid
  • login
  • new-game
  • play
  • register
  • status

Two supplementary endpoints exist for special-purpose use:

  • jsonp
  • old-game

Each endpoint accepts two HTTP request methods: POST and GET. If you click on any of the endpoint links above, your web browser will perform an HTTP GET request, which the endpoint interprets as a request for documentation. Follow the instructions provided by the endpoint in its documentation, and perform an HTTP POST request with your bot to interact with trigger the endpoint's functionality.

The first special-purpose endpoint, jsonp, exists to permit JavaScript bots to call API methods via HTTP GET requests. This is necessary due to a security scheme built into all modern web browsers, which JavaScript is subject to.

For examples of how to use the API, please see the example bots in the SDK. Each bot implements the most basic algorithm possible that will play the game. To be specific, if you run any bot in the SDK right now it will actually play the game, live, on the server.
