API Method :: PLAY

You have requested this page using the GET HTTP request method, to actually invoke this API endpoint you are required to use POST.

All JSON in this page has been formatted with JSONLint, whitespace is not significant to the server. The order of keys in JSON object doesn't matter.

This method permits you to play a card, ending your turn in the current game you are playing. This method accepts JSON in the body of the HTTP request, as follows:

{
    "session": "A hex string will appear here."
    "card": "H05"
}

The card that you play must be in the list of cards dealt to you in the response to calling the new-game method, and the card must not have been played in a previous round in the current game. Yes, the server checks for this.


Possible responses:

  1. The server will respond with the following if the card you played was was valid according to the game's rules:

    {
    "result": "success"
}

  2. The server will respond with the following if the card you played was not allowed:

    {
    "result": "failure",
    "reason": "An explanation of the failure will appear here."
}

It is important to remember that if you are playing the second card in the round, you must choose a card whose suit matches the first card that was played, unless you have no cards of that suit remaining. The server checks for this too.
