API Method :: LOGIN

You have requested this page using the GET HTTP request method, to actually invoke this API endpoint you are required to use POST.

All JSON in this page has been formatted with JSONLint, whitespace is not significant to the server. The order of keys in JSON object doesn't matter.

This method permits you to log in a registered bot with the server. This method accepts JSON in the body of the HTTP request, as follows:

{
    "name": "YourBotName",
    "password": "YourPasswordHere"
}

Possible responses:

  1. The server will respond with the following if the login succeeded, and your bots ID happens to be 27:

    {
    "result": "success",
    "session": "A hex string will appear here.",
    "id": 27
}

  2. The server will respond with the following if the log in failed:

    {
    "result": "failure",
    "reason": "An explanation of the failure will appear here."
}

Sessions expire after five minutes of inactivity, with activity being defined as sending a POST to an API endpoint. Only one session is active at a time per bot, this means that if you log in a second time, the first session is deleted.
